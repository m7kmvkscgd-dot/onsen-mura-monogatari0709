# 温泉村物語

`C:\ダンジョン1\` を複製して作った新プロジェクト。ゲームの仕組み(パーティ制・パーマデス・戦闘システム等)はダンジョン1から引き継ぎつつ、**和製(和風)の方向性に作り変えていく**ことが目的。ダンジョン1自体は元のまま残っており、このプロジェクトはそこから分岐した独立した作業ブランチ。

詳細な過去の作業履歴は docs/開発履歴.md を参照。

- CLAUDE.md が 30,000バイトを超えたら古い記録を docs/開発履歴.md へ移す。

**ユーザーが「せりふくん軽度ストレス生成」のようにこのシステムでのセリフ生成を求めるコメントをしたら、まず現在起動しているモデルを確認する。Fable 5ならそのまま`serifu/CLAUDE_serifu.md`のルールに従って生成に入ってよい。Fable 5でなければ生成せず、「現在Fable 5になっていません」等と伝えて指示を待つこと**(このシステムは品質最優先でFable 5前提のため、無断でSonnet 5等にフォールバックしない)。

## 次回このプロジェクトを再開する時の手順
1. `C:\温泉村物語\` を開く(このCLAUDE.mdが自動的に読み込まれる)。デスクトップの`温泉村物語 Claude Codeを起動.lnk`をダブルクリックすると、このディレクトリで`claude`が起動した状態のコマンドプロンプトがそのまま開く(コード上は`cmd.exe /k "cd /d C:\温泉村物語 && claude"`)。スマホからも指示を送りたい場合は`温泉村物語0711.lnk`(または既存の`温泉村物語 Claude Code(リモート)を起動.lnk`、中身は同じ)を使う。こちらは`claude --remote-control`付きで起動し、リモート操作(スマホ等)を有効にする
2. ローカルサーバーが必要な場合は `python nocache_server.py`(ポート8127)を`C:\温泉村物語\`で起動、またはデスクトップの`温泉村物語v2.3を起動.lnk`をダブルクリックすると`index.html`を直接ブラウザで開ける(file://起動、サーバー不要。ただしlocalStorageのセーブは同じ起動経路を使い続けないと引き継がれない点に注意)
3. スマホ実機で確認したい場合はCloudflareトンネルを新規に立てる必要がある(トンネルは使い捨てで、前回のURLは失効している)。「スマホで見たい」と伝えれば都度セットアップする。**あるいは本番URL`https://onsen-mura-monogatari.pages.dev`(下記「Cloudflare Pages本番デプロイ」参照)をそのままスマホのSafariで開いてもよい**(常時稼働・トンネル不要)
4. 作業を続ける際は本ファイル(CLAUDE.md)の「バージョン履歴」と各見出しの実装メモを読めば経緯を追える。gitログ(`git log --oneline`)も各コミットが1機能単位でまとまっている

## Cloudflare Pages本番デプロイ(重要な運用ルール)
このプロジェクトは`https://onsen-mura-monogatari.pages.dev`にCloudflare Pagesとして常時公開されており、ユーザーは主にこのURLを実機Safariで開いて動作確認する(トンネルより優先度が高い確認手段)。デプロイは以下のコマンドで行う:
```
npx wrangler pages deploy . --project-name=onsen-mura-monogatari --branch=main --commit-dirty=true
```
**`--branch=main`を必ず明示すること。** このオプションを省略すると、wranglerはgitの現在のローカルブランチ名(このリポジトリでは`master`)を使ってデプロイしてしまい、Cloudflare Pages側のProduction環境は`main`ブランチにひも付いているため、`--branch`未指定だとPreview環境にしか反映されず本番URLが更新されない(実際にこれで「直したのにスマホでは直っていない」という事故が一度発生した)。
デプロイ後は必ず`curl -sL https://onsen-mura-monogatari.pages.dev/index.html`等でファイル内容を取得し、ローカルの該当ファイルと`diff`で完全一致することを確認してから完了報告すること(SPAフォールバックにより存在しないパスでも200が返るため、ステータスコードだけでは正しくデプロイされたか判別できない)。本番エイリアスURL(`onsen-mura-monogatari.pages.dev`)は個別デプロイURL(`https://<hash>.onsen-mura-monogatari.pages.dev`)よりCDNキャッシュの反映が数秒〜数十秒遅れることがあるため、diffが一致しない場合は15〜20秒待って再確認する。

## ダンジョン1から引き継いだ仕組み(ゲームエンジン部分は共通)
- 4人パーティ制、ターン制バトル
- MP制(魔法系だけでなく物理必殺技もMP消費。町でのみ回復)
- 疲労度システム(潜り続けると弱体化、町で回復)
- パーマデス+瀕死救出システム(戦闘不能になると死体ではなく瀕死になり、別の仲間がその階に到達して救出コマンドで連れ帰れる。詳細は下記「瀕死・救出システム」参照)
- 複数敵・範囲攻撃(1〜3体の集団が出現し、範囲技持ちが雑魚の群れに有効)
- 職業ごとの武器/防具(3段階、レベルで解禁、購入すると同職業全員に永久ステータス加算)
- ダンジョン探索は「進む」ボタンのみの線形進行(方向選択・グリッド移動なし)
- スマホ前提のレイアウト

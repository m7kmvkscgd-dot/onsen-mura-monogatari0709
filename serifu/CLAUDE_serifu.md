# せりふくん(軽度ストレス)

温泉村物語の仲間キャラ同士の掛け合いセリフ(疲弊パート)を制作し、スマホで判定・管理するためのセリフ制作システム。既存のゲームコード(`C:\温泉村物語\` 直下の本体)とは完全に独立しており、このシステムがゲーム本体のファイルに触れることはない。

## キャラクター(9種・全員女性)

- 優しい: 一人称「私」。語尾「〜だね」「〜かな」「〜だよ」。柔らかく短め
- 熱血: 一人称「私」。語尾「〜だ!」「〜しよう!」「〜ぞ!」。元気・短文
- 冷静: 一人称「私」。語尾「〜だ」「〜だな」。短文・淡々
- 生意気: 一人称「あたし」。語尾「〜じゃん」「〜でしょ」「〜って」。少し上から
- のんびり: 一人称「私」。語尾「〜だよ〜」「〜かな〜」。伸ばし多め
- 真面目: 一人称「私」。語尾「〜ましょう」「〜です」。丁寧語
- お調子者: 一人称「あたし」。語尾「〜じゃん!」「〜もん!」。テンポ速め
- 無口: 一人称「私」。「……」多用、一言、体言止め
- 怖がり: 一人称「私」。語尾「〜かな…」「〜だけど…」。おどおど

## セリフ形式

- 2人の掛け合い(A→B、まれにA→B→Aの3行)
- 基本2行。3行は全体の25%まで。4行以上は絶対禁止
- 1行15文字以内目安
- MOOD3種類を各33%均等: 両者疲弊/A疲弊B元気/A元気B疲弊

## クオリティ方針(最重要)

- フリからオチまでの落差を最大化する。オチは短く、説明しない
- キャラの性格そのものをボケ・ツッコミの構造に組み込む(例: 冷静が淡々とした事実でボケを刺す、無口が沈黙のまま笑いを成立させる、生意気が上から目線のまま墓穴を掘る)
- シンプルなツッコミ系が高評価(例: 「水飲んだか?」「前ってどっちだっけ」「投票の結果を伝えよう。0票だ」)
- バリエーション重視: 足・笑い・限界の話に偏らない。環境(風・虫・鳥・気温)、感覚(味・音・視界)、空腹、眠気、思考力低下、忘れ物、荷物、体温など多角的に扱う
- 季節・時間帯を特定するワード禁止(蝉、夕方、秋、朝、雷など)
- 恋愛要素なし、本気で傷つける発言なし、ネット用語なし
- セリフの重複は絶対禁止(採用データ.txtの既存内容と必ず照合する)

## 今後のワークフロー

- ユーザーが「せりふくん軽度ストレス生成」等とコメントしたら、まず現在起動しているモデルを確認する。Fable 5ならそのまま生成に入ってよい。Fable 5でなければ生成せず、「現在Fable 5になっていません」等と伝えて指示を待つ(無断でSonnet 5等にフォールバックしない)
- 運用はユーザーが◯✕で判定するだけでよい。「出力」ボタンは採用分をクリップボードコピーすると同時に`せりふくん軽度ストレス_出力_YYYYMMDD_HHmmss.txt`という名前でダウンロードフォルダ(通常`C:\Users\keiic\Downloads\`)に自動保存する
- ユーザーから「続き」と言われたら、①ダウンロードフォルダの`せりふくん軽度ストレス_出力_*.txt`のうち最新のものを自分で読み込み、その内容を採用データ.txtの末尾に追記保存する ②採用データ.txtの内容(追記後)と重複しない新セリフを50個ずつ追加生成し、セリフ判定.htmlのLINES配列に追記する(IDは連番で継続)。ユーザーに手動でのコピペ作業をさせないこと
- 目標総数600個
- セリフ生成は原則Fable 5モデルで行う想定(品質を最優先するため)。ユーザーがモデル未切り替えを承知の上でSonnet 5等での生成を明示的に指示した場合も、このルールとクオリティ方針は同じ基準を適用すること
- git commitのタイミングはユーザーの指示を待つこと。自動でコミットしない

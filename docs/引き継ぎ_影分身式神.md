# 引き継ぎ: 忍者「影分身の術」/陰陽師「式神召喚」(2026-07-21夜)

深夜の指示で実装した「一時的な5人目の戦闘要員」システムについて、次回セッション開始時にまず確認してほしい判断・懸念事項をまとめる。実装自体はコミット済み・本番デプロイ済み・Node vm検証とPlaywright実ブラウザ検証の両方でパスしている(詳細はコミット`73a5476`のメッセージ参照)が、**実機での手触り確認(スマホでの実プレイ)はまだ行っていない**。このセッションではClaude in Chrome拡張が接続されておらず、ヘッドレスのPlaywrightで機能面だけ検証した状態。

## 何を作ったか(1行まとめ)

- 忍者「影分身の術」(tier4右、4MP): HP75%/MP0の自分の分身を隣に呼び出す。通常攻撃のみ、状態異常無効、回復不可、瀕死にならず即消滅、戦闘終了で自動消滅
- 陰陽師「式神召喚」(tier6右、6MP。旧プレースホルダー「六芒陣」を差し替え): 自律AIで戦う式神を呼び出す。回復可能、戦闘をまたいで持続、消滅は撃破/野営/茶屋一休み/「式神帰還」コマンドのいずれか
- 両者とも「5人目の枠」は同時に1つまで(片方が出ている間はもう片方を呼べない)

## ユーザー確認が必要な判断(仕様に明記が無く、自分の判断で決めた箇所)

1. **影分身を唱えるコスト自体は4MPのまま**(元のプレースホルダーの値を流用。分身自身のMPが0という指定はあったが、召喚コスト自体の数値指定は無かった)
2. **影分身のdef/magも本体からコピー**(仕様で名指しされたのはatk/spdのみだが、防御力0だと存在意義が薄いと判断し、本体の素の値をそのまま4ステータス全部コピーする形にした。装備ボーナス(equipBonus)は分身にはコピーしていない)
3. **影分身の表示名は「(本体の名前)の分身」**(例: 「はるかの分身」)。UI上の区別のためだけの命名で、変更は容易
4. **影分身のHP75%は「本体の最大HP」の75%**(本体の現在HPではない。瀕死寸前で唱えても分身はフルHPで出る)
5. **式神召喚のMPコストは6**(以前一時停止していた設計案の初期案をそのまま採用。式神本体のステータス式も同じ設計案を流用: `maxHp=(14+mag*0.5)*成長率、atk=(4+mag*0.3)*成長率、def=3固定、spd=13*(1+level*0.05)`。全て仮の値、本物の絵と一緒に再調整する前提)
6. **「式神帰還」コマンドの置き場所**: 戦闘中は行動メニューに「式神帰還」ボタンとして追加(変身解除と同じ、MP・ターン消費0で押した後も続けて行動選択できる)。探索中は画面下部の常設ボタン欄(`dungeonPotionBtn`等が並ぶ場所)に「式神帰還」ボタンを追加。この配置は完全に自分の判断
7. **副作用: 「六芒陣」の魔力+15%コンボ効果が失われた**。式神召喚を旧「六芒陣」(tier6右)に上書きしたため、「結界術」(tier3右)を使った直後の魔力+15%ボーナスが無くなった。同じコンボの防御力+15%側(tier9右「霊脈支配」)は無傷で残っている。これを何かで埋め合わせるかは未対応、ユーザー判断待ち
8. **状態異常免疫のバグ修正**: `resolveDebuffEffect`(敵の通常攻撃が状態異常を付与する経路)が、poison/bleed/burn/stun/silenceはstatusImmuneTurnsをチェックしていたのに、atkDown/defDown/spdDown/dmgTakenUpの4種だけチェックが抜けていた。影分身の「状態異常にならない」を正しく機能させるため、この4種にもチェックを追加した。これにより既存の僧侶「天恵の祈り」(状態異常無効5ターン)にもこの4種が今後正しく効くようになる(意図した副次効果)
9. **陰陽師本人が瀕死/離脱した場合の式神の扱いは未対応**。今回の指示にも言及が無く、現状は式神がそのまま自律的に戦い続ける。不自然に見える場合は「オーナー不在なら式神も強制送還」等の追加ロジックが必要
10. **セーブ/再開でfieldPartyが再構築される問題は未解決**(以前から分かっていた既知の課題)。影分身/式神はゲーム内の一時オブジェクトで`state.roster`には一切登録されないため、同じ遠征中の連戦・野営などライブセッション内では問題なく持続するが、**アプリを閉じて後で再開する(セーブ→リロード)と、影分身/式神は黙って消える**。式神の「戦闘をまたいで持続」という仕様は「同じセッション内では」という条件付きで実現している状態。[[user-onsen-phone-remote-only]]のような「アプリを閉じて後で再開」という使い方と相性が悪い可能性がある

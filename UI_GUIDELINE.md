# 温泉村物語 UIデザインガイドライン

2026-07-10のUI/UX全画面監査(39項目, S8/A18/B13)を踏まえて策定。既存の`index.html`内`--dw-*`トークンの正式な仕様書として、以後の実装・改修はこの基準に統一する。

## 1. 余白ルール
- 画面左右の余白: 16px
- 画面上下の余白: 14px
- カード同士の間隔: 12px(`--dw-space-card-gap`)
- セクション間の余白: 20px(`--dw-space-section`)。見出しが変わる境目はこれ未満にしない
- 区切り線の上下: 16px(出発準備画面の3ブロック分離で採用済み)

## 2. 文字サイズルール
| 役割 | サイズ | 色 |
|---|---|---|
| 画面タイトル | 22px / 700 | `#f3e7c6` |
| 見出し(h2) | 18px / 700 | `#ead8ab` |
| 本文 | 15px / line-height 1.5 | `#ece8dc` |
| 補足・キャプション | 13px | `#b6b0a2` |

説明文は2行以内を目安にする。3行を超える場合は要約するか折りたたみ(増築画面の方式)を検討する。

## 3. カードデザインルール
- 背景: `#2f3540`
- 枠線: `#5d4d35` / 2px
- 角丸: 14px(主要カード) / 8px(行・小カード) / 6px(サムネイル)
- 影: `0 6px 18px rgba(0,0,0,.28)`
- padding: 上下14px・左右16px

**役割に応じた特別カード**: 「主役」(キャラクターポートレート)は通常カードの背景をそのまま使わず、和柄の枠飾りかクラスカラーの微光を足す。「書状」(奉行所の依頼)は和紙テクスチャ+墨文字調タイトルを足す。全てを同じ`.card`で済ませない。

## 4. ボタンルール
- 高さ: 48px(min-height)
- 角丸: 12px
- 文字: 15px / 600
- 通常背景: `#7b5d36` / hover: `#937148`
- 押下: `scale(.97) + brightness(.88)` / 100ms
- 影: `0 3px 10px rgba(0,0,0,.25)`

**色の意味付け**: 金=推奨/主要行動、茶=通常行動、赤=取り返しがつかない操作のみ(解雇等)。「逃げる」「里に戻る」のような中立的な離脱操作は、赤ではなく専用の琥珀トーンで区別する(現状はどちらも赤で意味が薄まっている、要修正)。

**押せない理由の伝え方**: ボタンを無効化するだけでなく、タップ時に理由をアラートで伝える(奉行所の受注ボタンで確立した鉄則)。温泉・宿屋の「押せないボタンが並ぶだけに見える」問題にも同じ考え方を適用する。

## 5. アイコンルール
絵文字を新規箇所で使わない。ステータス画面で導入した`currentColor`ベースの線画SVG(`STATUS_STAT_ICONS`)を唯一のアイコン言語とし、今後は金・回復薬・煙玉・施設種別などもこのセットへ拡張する。

サイズは文脈に応じて2段階:
- 通常16px(戦闘の状態異常等、密度が高い場所)
- 強調24px(ステータス画面などアイコン自体が主役の場所)

横間隔は常に6px。

## 6. 色の使い分け
| 用途 | 色 |
|---|---|
| 金(通貨・経験値・主要CTA) | `#d7b66f` |
| 成功・HP | `#6ea95b` |
| 危険(取り返しがつかない操作のみ) | `#c95d54` |
| 情報・MP | `#7b93b9` |
| 説明・補足 | `#9d998e` |

この5色以外の新規アクセント色を増やさない。既存の「NEW」バッジ(`#ff6a4d`)は6色目の逸脱のため、危険色か金色の変種に差し替えを検討する。

## 7. アニメーション速度ルール
| 用途 | 速度 |
|---|---|
| ボタン押下 | 100ms |
| カード展開/スライドイン | 180ms |
| 画面遷移 | 220ms(opacityのみ、position:fixed要素の基準を壊さない) |
| ダメージ演出 | 180ms |
| 会心・敵出現 | 250ms |
| 勝利/敗北の専用演出(新規) | 400〜600ms(唯一の例外、「間」を作るための意図的な長さ) |

受注スタンプ・会心の複数ステップ演出のように、意図的に作り込まれた一連の物語的シーケンスはこの一覧の対象外(総尺を無理に合わせない)。

## 8. 画面遷移ルール
通常の画面切替は220msのopacityフェードで統一(実装済み)。「勝利」「敗北」「レベルアップ」など感情的なピークを伴う遷移だけは、フェードの前に専用の一枚(バナー/暗転)を挟む2段階構成にする。1段階目(結果の告知)と2段階目(詳細情報)を必ず分離し、告知なしで詳細画面へ直接遷移させない。

## 9. 情報配置ルール(視線誘導)
全画面で「①タイトル→②最重要情報→③操作」の3ステップを画面上から下へ一直線に並べる。最重要情報は画面ごとに1つに絞る(村なら所持金、依頼画面なら受注可否、戦闘なら次に取るべき行動)。

同格に見える選択肢を並べる時は最大4つまでとし、それを超える場合は最初にグルーピングを検討する。同じ内容のカードが5枚以上連続する画面は、必ずグルーピングかタブ分割を検討する(増築画面の15施設、宿屋の名簿+雇用など)。

## 未着手のS評価(優先度順)
1. 勝利/敗北が同一テンプレート(演出の作り込み差が最も出る場所)
2. 敗北専用の演出が存在しない(暗転+専用文言+専用配色が必要)
3. 施設一覧が視線誘導のない棚卸しリスト(建築可能/建築済み/条件未達の3グループ化)
4. 「🏠家」だけ折りたたみ仕様から外れている一貫性の欠如
5. 温泉のストレス0時、押せないボタンが並ぶだけに見える(奉行所と同じ事故の再発リスク)
6. アイコン言語が絵文字とSVGで混在
7. 敵絵とキャラ絵の画風不一致(アート方針の問題、UI側では額装で緩和)
8. 村の日付/時間/所持金が背景美術に負けて最重要情報として機能していない
